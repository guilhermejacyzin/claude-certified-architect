# Curso Completo - Claude Certified Architect Foundations

Este é o caminho principal do curso. A proposta é formar do básico ao nível
especialista, com foco em aprovação na certificação e aplicação em projetos
reais.

## Como o curso está organizado

| Nível | Arquivo | Objetivo |
|---|---|---|
| 0 - Orientação | `00-como-usar-o-curso.md` | Entender método, rotina e critérios |
| 1 - Fundamentos | `01-fundamentos.md` | Dominar vocabulário e ideias centrais |
| 2 - Praticante | `02-praticante.md` | Construir soluções simples com tools, MCP e Claude Code |
| 3 - Arquiteto | `03-arquiteto.md` | Desenhar sistemas agentic completos |
| 4 - Especialista | `04-especialista.md` | Tomar decisões de produção, confiabilidade e escala |
| Avaliação | `05-avaliacoes-rubricas-capstone.md` | Medir prontidão e executar projeto final |

## Resultado esperado

Ao final, a pessoa deve conseguir:

- explicar os cinco domínios da prova;
- projetar agentic loops;
- desenhar MCP tools/resources/prompts;
- configurar Claude Code para time;
- criar structured output com validação;
- gerenciar contexto e proveniência;
- decidir quando usar humano no fluxo;
- responder questões scenario-based com raciocínio arquitetural;
- construir um projeto final simulando uma solução de produção.

## Carga horária sugerida

| Ritmo | Duração | Uso recomendado |
|---|---:|---|
| Intensivo | 7 a 10 dias | Quem já tem experiência com LLMs e código |
| Normal | 3 semanas | Melhor equilíbrio para estudo e prática |
| Profundo | 4 a 6 semanas | Para formar turma e nivelar diferentes perfis |

## Regra de progressão

Não avance de nível só por leitura. Avance quando conseguir:

1. explicar o conceito sem olhar;
2. resolver exercícios;
3. responder questões simuladas;
4. criar um exemplo próprio;
5. apontar pelo menos dois anti-patterns.

## Método de estudo recomendado

Para cada aula:

1. Leia a explicação normal.
2. Leia a explicação técnica.
3. Reescreva o exemplo simplificado com suas palavras.
4. Faça a atividade.
5. Responda 3 a 5 questões relacionadas.
6. Registre lacuna no `../study-log.md`.

## Critério de especialista

O nível especialista não significa decorar todos os termos. Significa conseguir
projetar uma solução completa justificando:

- quais partes ficam no modelo;
- quais partes ficam em tools;
- quais partes ficam em hooks/código;
- quais partes exigem schema;
- quais partes exigem revisão humana;
- como o sistema mantém contexto, fonte e auditoria.
